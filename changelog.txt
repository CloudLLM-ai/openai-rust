1.7.1 - 2025-12-04
 - new: xAI Responses API support (/v1/responses endpoint) for agentic tool calling
 - new: ResponsesArguments struct with input (messages) and tools fields
 - new: ResponsesMessage struct for Responses API input format
 - new: ResponsesCompletion, ResponsesOutputItem, ResponsesContent, ResponsesUsage response types
 - new: Client::create_responses() method for calling the Responses API
 - new: ResponsesCompletion::get_text_content() helper to extract text from response
 - fix: grok_tools field now serializes as "server_tools" for Chat Completions API compatibility
 - See: https://docs.x.ai/docs/guides/tools/search-tools

1.7.0 - 2025-12-04
 - BREAKING: Removed deprecated search_parameters and SearchParameters/SearchMode (xAI Live Search API deprecated Dec 15, 2025)
 - new: grok_tools field in ChatArguments for xAI Agent Tools API (serializes as "tools")
 - new: GrokTool struct with factory methods: web_search(), x_search(), code_execution(), collections_search(), mcp()
 - new: GrokToolType enum: WebSearch, XSearch, CodeExecution, CollectionsSearch, Mcp
 - new: GrokTool builder methods: with_allowed_domains(), with_date_range()
 - new: ChatArguments::with_grok_tools() builder method
 - See: https://docs.x.ai/docs/guides/tools/overview

1.6.0 - 2025-05-27
 - new: added search_parameters chat argument proposed by Grok with SearchMode enum (on | off | auto), return_citations, and date-range fields (from_date, to_date)

1.5.9 - 2025-05-16
 - new: support for gpt-image-1 model in image generation via /v1/images/generations endpoint
 - new: Base64 decoding in tests.rs to save generated images to generated_image.png
 - new: base64 crate (version 0.22.1) as a dependency for decoding image data
 - new: chat::ImageGeneration struct and field in chat.rs for future chat-based image generation support
 - new: Detailed prompt in create_image test for high-quality, photorealistic output
 - Removed response_format parameter from ImageArguments in images.rs, as gpt-image-1 always returns base64-encoded images
 - Updated create_image in lib.rs to use gpt-image-1 with auto quality and no response_format
 - Changed size field in ImageArguments to Option<String> to support "1024x1024", "1024x1536", and "1536x1024"
 - Updated test models in tests.rs from text-davinci-003 to gpt-3.5-turbo-instruct for compatibility
 - Bumped dependency versions: anyhow to 1.0.98, reqwest to 0.12.12, serde to 1.0.218
 - Updated repository URL in Cargo.toml to https://github.com/cloudllm-ai/openai-rust
 - Improved documentation and removed outdated comments in chat.rs, images.rs, and lib.rs
 - Notes: gpt-image-1 requires organization verification (government ID and facial verification). Ensure your OpenAI account is verified
 - Run client.list_models(None).await to confirm gpt-image-1 availability
